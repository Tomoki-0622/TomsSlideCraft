## 1. ビジネス背景・課題

サントリーホールディングス株式会社 デジタル&AI本部 グローバルITG では、以下の課題を抱えている：

- **プレゼン資料の作成に時間がかかりすぎる** — ストーリー設計からスライド完成まで多大な工数を要している
- **英語化（翻訳）に時間がかかる** — 日本語で作成した資料を英語版に変換する作業が別途発生している

---

## 2. 解決目標（成功基準）

| 指標 | 目標値 |
|---|---|
| プレゼン作成時間の削減率 | **70%削減** |

## 4. スコープ

### MVPに含めるもの

- AIチャットによるプレゼンストーリー設計
- Marpを使ったスライド生成・プレビュー・編集
- HTML / PPTX ダウンロード
- **日→英 全スライド翻訳（翻訳結果を新規プロジェクトとして保存）**
- **スクリプト作成（日本語・英語対訳、スライドサムネイル連動）**
- 共有パスワード認証
- ブラウザ LocalStorage によるプロジェクト保存

### MVPに含めないもの（将来拡張候補）

- サーバーサイドDB・複数デバイス対応
- 個人アカウント管理
- チーム共有・コメント機能
- スライドデザインテーマの追加
- 画像アップロード
- IndexedDBへのフォールバック（LocalStorage 5MB超過対応）
- LocalStorageデータの一括エクスポート・インポート（デバイス間移行用）

## 6. 非機能要件

| 項目 | 要件 |
|---|---|
| **AIレスポンス方式** | 全文生成完了後に一括表示（ストリーミングなし） |
| **データ保存先** | ブラウザ LocalStorage（上限5MB。超過時はユーザー通知） |
| **AIバックエンド** | Azure OpenAI（社内環境）。環境変数で接続先を管理 |
| **認証** | 共有パスワード1つ（httpOnly Cookie、有効期限7日） |
| **デプロイ** | Vercel（AI呼び出しRoute Handlerには `export const maxDuration = 60` を設定し最大60秒に延長。Marp/PPTX生成などの重い非AI処理はクライアント側で実行） |

## 7. 前提条件

- 利用者は Google Chrome または Microsoft Edge の最新版を使用できる
- 利用者はインターネット接続環境を持つ
- Azure OpenAI のデプロイは MVPリリースまでに利用可能になる（デプロイ名は決定次第 `AZURE_OPENAI_DEPLOYMENT` 環境変数に設定）
- Suntoryロゴ画像（`public/suntory-logo.png`）の使用は法務・広報部門の許諾を得たものとする

---

## 8. データガバナンス方針

- Azure OpenAI（社内環境）を使用するため、プレゼン内容が外部に送信されるリスクは低い
- ただし、以下の注意事項をバナーで表示する：

> 「本ツールで入力した内容は社内Azure OpenAI環境に送信されます。社内情報取り扱いルールに従ってご利用ください。」

**バナーの表示タイミング**:
- LocalStorageに `slidecraft:governance_accepted` フラグが存在しない場合のみ、ログイン後のダッシュボード上部に表示する
- ユーザーが「確認しました」ボタンをクリックすると `slidecraft:governance_accepted: true` をLocalStorageに保存し、以降は表示しない（**生涯1回のみ表示**）
- ブラウザ・端末を変えるとLocalStorageがリセットされるため再表示される（許容範囲とする）
